Onboarding = walkthrough introducing a user to the app on first launch. and != inviting a another user.

Asking to pray:

  • Does this function as inviting users without the app?
  • During the onboarding process after a user first installs the app?
  • During the request to follow interaction? how is this triggered?
  • does this trigger when a request to pray(follow) is initiated, which would include users with the app?

Screen for own prayers:

  • do we only show a list of own prayers on the users own profile screen?
  • do we need a tab for this?
  • when someone asks you how they can pray for you, do we force a new prayer to be written? do we show the user a screen with a list of prayers where he/she may select a prayer to send?
Flow?
  • User receives notification of other user asking for a prayer
  • User taps the notification in notification screen
  • App moves to (a new) view of own prayers
  • User can either select one, or create a new prayer

*Problem: prayer is only sent to the requesting person if a new prayer is created. Assumes prayers have visibility/privacy features.

IMO, "Asking to Pray" should be a feature used when *inviting* external users to the app, not a feature triggered by a follow request.

Prayer functions:

  • Do we allow prayers to be threaded to other prayers?
  • Are we keeping commenting?
  • Are we keeping answering?
  • What functions does a prayer have? Thread another prayer? Comment? Answer? Delete? Edit? Set visibility/privacy?

For MVP, I'd suggest launching without some of these features

Are we letting users style their prayers? From meeting with camera, we decided to let users select an app “theme” which would dicate how all their incoming prayers look. Patterns? Gradients? Images?

Notifications:

  • Do we need a notifications tab?
  • How many different types of notifications will the app have?
  • Recieving a request? Another user adding your prayer? A prayer on your list is answered, concatenated, closed, commented?
  • Do we need to separate these notification types into separate tabs?
Main Views:

  • Other (sidebar)
    • Notifications

    • Account
    • Logout
  • (Potential) Own Prayers
  • (Potential) Notifications

  • Own Profile becoms another main view, and would function as the "My Prayers" holder
  • Notifcations view as a main tab
  • Connect View as a main tab

*Current navigation works because of only 3 main views and the rest not accessible through the top nav. More main views will require a different navigation style
